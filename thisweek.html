This week we covered react and express, so I wanted to take a deeper look into react. I wanted learn more about it so I looked at the react website where they have guides for learning. The guide suggests starting with a static model of your app, creating a component hierarchy, and using prop instead of state. Their example for a app to check the stock of a store uses prop and their static code segment (segment 1).
Next they show an example of how to implement state objects. They give a list on how to identify where state objects should be used:
  • Is it passed in from a parent via props? so, it probably isn’t state.
  • Does it remain unchanged over time? If so, it probably isn’t state.
  • Can you compute it based on any other state or props in your component? If so, it isn’t state.
This is where it started to get confusing but it showed how to have the states update down the component hierarchy but when it came to updating the page based on the user input there was a lot to unpack.
